How clients will use My Aibud:
Client journey
1. Client signs up:
Visits your website (e.g., myaibud.com)
Clicks "Get Started"
Signs up with email/password (Clerk handles this)
System creates a tenant account with Starter plan (400 credits)
2. Client connects WhatsApp:
Goes to Settings page
Clicks "Connect WhatsApp"
Scans QR code with their phone
WhatsApp connected
3. Client uses the system:
Sends polls to leads via dashboard
Leads respond via WhatsApp
AI processes responses automatically
Credits deducted per message sent
4. Client manages account:
Views contacts and messages
Checks credit usage
Upgrades subscription if needed
Multi-tenant architecture
One deployment serves all clients
Each client = 1 tenant in the database
Each tenant can have multiple WhatsApp numbers (instances)
Data is isolated per tenant
Credits tracked per tenant
For you (platform owner)
Deploy once:
Deploy to Vercel (or any Node.js host)
Set production environment variables
Deploy Evolution API (or use managed service)
All clients use the same app
Monitor clients:
Query database to see all tenants
View usage per client
Manage subscriptions
Support tools (view client data)
Scaling:
Add clients without code changes
Each client gets their own WhatsApp instance
Credits automatically tracked
Billing can be integrated with Stripe

Quick deployment steps:
Create VPS (5 minutes)
Go to https://hetzner.com
Create Cloud Server: CPX21 (4 vCPU, 8GB RAM)
Choose Ubuntu 22.04
Cost: €6-12/month
Install Docker (5 minutes)
   curl -fsSL https://get.docker.com -o get-docker.sh   sh get-docker.sh   apt install docker-compose git -y
Deploy Evolution API (10 minutes)
   git clone https://github.com/EvolutionAPI/evolution-api.git   cd evolution-api   cp env.example .env   # Configure .env with production settings   docker-compose up -d
Configure Domain + SSL (15 minutes)
Point domain to VPS IP
Install Nginx + Let's Encrypt
Configure reverse proxy
Update My Aibud (5 minutes)
Change EVOLUTION_API_URL to your cloud domain
Update webhook URL in Evolution API
Total setup time: ~40 minutes
Monthly cost: $7-13 for 5-10 users
Scaling strategy
Phase 1: 1-10 users
1 VPS (Hetzner CPX21)
Cost: $7-13/month total
Phase 2: 11-50 users
2 VPS + Load Balancer (Nginx)
Cost: $14-26/month total
Distribute users across servers
Phase 3: 50+ users
Auto-scaling VPS cluster
Cost: $0.70-1.30/user/month
Add VPS when >80% capacity
Cost optimization tips
Instance sharing: 1 VPS can handle 10-20 WhatsApp instances
Resource limits: Set Docker limits to prevent overuse
Shared database: Use Supabase free tier initially
CDN for media: Use Cloudflare R2 ($0.015/GB)
Monitor usage: Track CPU/RAM, scale when needed
Stop local Docker
To stop Evolution API locally:
cd evolution-apidocker-compose down
This stops all containers and frees up resources on your laptop.
