Monthly maintenance checklist
Weekly tasks (15-30 minutes/week)
1. Monitor system health
Check Inngest dashboard for failed jobs
Review Sentry for errors
Check Evolution API logs for issues
Verify WhatsApp connections are active
2. Check resource usage
VPS CPU/RAM usage (should be <80%)
Database size (Supabase free tier: 500MB limit)
Bandwidth usage
Google AI API quota usage
3. Review failed jobs
Check failed_jobs table in database
Retry or investigate failures
Fix any recurring issues

Tools to reduce maintenance
1. Monitoring services (free/cheap):
UptimeRobot (free: 50 monitors)
Better Uptime (free tier)
Sentry (free tier: 5k events/month)
2. Automated backups:
Supabase: Automatic
Evolution API: Cron job for daily backups
Store in S3/Cloudflare R2 ($0.015/GB)
3. Auto-scaling:
VPS auto-scaling scripts (when CPU >80%)
Database auto-scaling (Supabase handles this)
4. CI/CD:
GitHub Actions (already set up)
Auto-deploy on git push
Run tests before deploy
What breaks if you skip maintenance
Low risk (can wait):
Minor Evolution API updates (security patches are important)
Dependency updates (unless security-related)
Performance optimization
High risk (do not skip):
Security updates (VPS, Docker, system)
Database backups (data loss risk)
Monitoring alerts (miss critical issues)
SSL certificate renewal (auto, but verify)
Recommended maintenance schedule
Week 1: System updates + security patches
Week 2: Database review + backups
Week 3: Cost review + optimization
Week 4: Performance review + client support
Daily: Check alerts (5 minutes)
Weekly: Quick health check (10 minutes)
Monthly: Full maintenance (1-2 hours)
Bottom line
With proper setup:
Monthly time: 1-2 hours
Most tasks are automated
Focus on monitoring and updates
Minimal manual intervention
Without automation:
Monthly time: 3-5 hours
More manual checks needed
Higher risk of missing issues
Recommendation: Set up monitoring alerts (Sentry, UptimeRobot) to reduce manual checks to ~1 hour/month.
